CRITICAL TECHNICAL DETAILS
================================================================================

PROFILE LAYOUT SYSTEM:

Database Field:
  User.profileLayout = {
    leftZone: ['music', 'topfriends'],
    rightZone: ['assets', 'socialhub'],
    bottomZone: ['posts'],
    mobileOrder: ['music', 'topfriends', 'posts', 'assets', 'socialhub']
  }

Frontend Files:
  - ProfileBuilder.tsx        → Drag-drop editor component
  - ProfileDesignModal.tsx    → DESIGN modal (has LAYOUT tab)
  - ProfileWidgetRenderer.tsx → Maps widget IDs to React components
  - ProfileGrid.tsx           → Renders widgets based on layout

Backend Files:
  - backend/routes/users.js   → PUT /users/profile endpoint
  - backend/models/User.js    → profileLayout field in schema

Save Flow:
  1. User drags widget in ProfileBuilder
  2. Clicks SAVE LAYOUT
  3. Frontend calls: userAPI.updateProfile({ profileLayout: layout })
  4. Backend receives PUT request, saves to MongoDB
  5. onProfileUpdate callback fires
  6. ProfileGrid re-renders with new layout
  7. Layout persists on refresh

Available Widgets:
  - music        → MusicPlayerWidget
  - topfriends   → TopFriendsWidget
  - socialhub    → SocialHubWidget
  - geonode      → GeoNodeWidget
  - assets       → AssetGalleryWidget
  - datalog      → DataLogWidget
  - customcode   → CustomCodeWidget (stub)
